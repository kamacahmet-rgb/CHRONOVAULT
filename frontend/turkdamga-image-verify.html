Üç Katmanlı Görsel Analiz

Katman 1
SHA-256 Tam Eşleşme

Dosyanın her biti aynıysa eşleşir. Birebir aynı dosya, farklı isimle yükleme veya indirip tekrar yükleme durumlarını anında tespit eder. O(log n) hız, B-tree index.

Sınır: Tek piksel değişikliği, JPEG yeniden kaydetme veya yeniden boyutlandırma farklı hash üretir.

Katman 2
pHash — Algısal Hash

64-bit perceptual hash. DCT (Discrete Cosine Transform) tabanlıdır. Görsel içeriğin "özünü" yakalar. Yeniden boyutlandırma, hafif renk düzeltmesi, JPEG sıkıştırma, küçük watermark ekleme gibi değişikliklere karşı dayanıklıdır.

Eşik: Hamming mesafesi ≤ 10/64 → benzer (%85+ aynı)

Katman 3
CLIP Embedding — Motif Analizi

OpenAI CLIP modeli (ViT-B/32) ile 512 boyutlu görsel vektör. Aynı halı motifinin farklı renk, ölçek veya perspektif versiyonlarını tespit eder. Vektörler PostgreSQL pgvector'de IVFFlat index ile saklanır, cosine similarity ile aranır.

Eşik: Cosine benzerliği ≥ 0.82 → güçlü motif benzerliği
Örnek: Kırmızı zemin üzerine siyah Kafkas motifi → mavi zemin üzerine beyaz aynı Kafkas motifini bulur.

Veritabanı & Index Yapısı
-- pgvector extension
CREATE EXTENSION vector;

-- Tablo
CREATE TABLE image_stamps (
  sha256      CHAR(64),        -- B-tree index
  phash_int   BIGINT,          -- bit XOR index
  embedding   vector(512),     -- IVFFlat index
  ...
);

-- SHA-256: B-tree (tam eşleşme, O(log n))
CREATE INDEX ON image_stamps(sha256);

-- pHash: bit operasyonu için
CREATE INDEX ON image_stamps(phash_int);

-- CLIP: yakın komşu arama
CREATE INDEX ON image_stamps
  USING ivfflat (embedding vector_cosine_ops)
  WITH (lists = 100);
